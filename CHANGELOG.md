## [Unreleased] — 1.3.0 — 2026-07-18

### Добавлено

- 📋 **Страница «Условия программы»** (`/rules`): ссылка из футера главной и страницы парка; пока содержит аккуратную заглушку будущего положения программы. Краткая памятка на странице парка переименована в «Краткие требования к активации»
- 👤 **Регистрация и авторизация активаторов** (кастомная, в стиле сайта): страницы `/register`, `/login` (вход по позывному или email), личный кабинет `/cabinet` со списком своих активаций и статусами модерации. Загрузка активаций теперь требует входа; позывной подставляется из профиля; активация привязывается к `user_id`. Роли `user`/`moderator`/`admin`, доступ в админку — только модераторам/админам
- ✉️ **Подтверждение email через Resend** (SMTP): письмо при регистрации, страница/баннер с повторной отправкой; загрузка активаций доступна только с подтверждённым email. Модераторы/админы помечаются подтверждёнными миграцией
- 🛠️ **Управление пользователями в Filament** (`UserResource`, только для админов): смена роли, пароля, подтверждение email, переименование, удаление (нельзя удалить себя), фильтр по роли, счётчик активаций
- 🌳 **Публичная страница активации** (`/activations/{activation}`): галерея фото с лайтбоксом, **журнал QSO** (позывные охотников, время UTC, диапазон, мода, RST) и сводка лога, скачивание ADIF (генерируется из сохранённых QSO). Фото публичны, скриншот QTHnow остаётся приватным. Ссылки на активацию с главной и со страницы парка (значок 📷 с числом фото). Гостям видны только одобренные активации, модератору — любые для предпросмотра
- 📋 **Страница всех парков** (`/parks`): карточки с группировкой по городам и числом активаций; пункт «Парки» в шапке ведёт сюда (раньше — якорь на секцию главной)
- 🗺️ **Карта-пикер координат в админке**: в форме парка (Filament) перетаскиваемый маркер MapLibre + клик по карте задают широту/долготу; ручной ввод в поля двигает маркер. Работает и в модалке «Просмотр» (read-only)
- 🔗 Кликабельные логотип и заголовок «Urban Parks» в шапке (ведут на главную); пункт «На сайт» в админ-панели Filament
- 🌍 Поля «Название (EN)» и «Описание (EN)» в форме парка Filament

### Изменено

- 🌍 `/api/parks` отдаёт `name`/`description`, локализованные по текущему языку (раньше английские описания в карточках парков и попапах карты оставались русскими)
- 🏆 «Столица программы» на главной считается динамически (город с наибольшим числом парков) вместо захардкоженного числа
- 📝 Секция на главной переименована в «Последние добавленные парки» / «Recently added parks», сортировка — по дате добавления
- 🌳 `ParkSeeder` наполнен реальными парками Новосибирска (корректные координаты, описания RU/EN, референсы `UP-RU-NSK-0001…0007`); идемпотентный `updateOrCreate` — повторный запуск не плодит дубли
- 🚫 Тестовые активации (`ActivationSeeder`) исключены из `DatabaseSeeder` — на боевом деплое сидируются только реальные данные

### Исправлено

- 🐛 Убрана нерабочая кнопка «Скачать ADIF парка» на странице парка (отдавала заглушки band/mode/time); корректный экспорт ADIF теперь на уровне активации из реальных QSO
- 🐛 Карта в модалке «Просмотр» парка (Filament) не отображалась и роняла ошибку Livewire (`PublicPropertyNotFoundException`) — исправлено: self-contained Alpine-компонент, инициализация координат из записи, read-only в режиме просмотра

## [Unreleased] — 1.2.0

### Добавлено

- 📤 **Загрузка ADIF-логов**: форма «Загрузить активацию» принимает .adi-файл + обязательный скриншот QTHnow + до 5 фото. Лог парсится, QSO пишутся в базу, активация уходит на модерацию (`ActivationImporter`, транзакция, защита от дублей, rate limit 10/час)
- 🔍 **Экран модерации в Filament**: страница просмотра активации со скриншотом/фото (private-диск, отдача только авторизованным), сводкой лога (диапазоны, моды, время работы UTC), кнопками Одобрить/Отклонить и скачиванием исходного .adi
- 🗺️ **Карты на MapLibre GL JS** (векторные тайлы OpenFreeMap вместо растрового OSM): плавный зум, кластеризация без плагинов, `cooperativeGestures` — колесо мыши скроллит страницу, зум только Ctrl+колесо (на мобильных — два пальца). Карта в контейнере ограниченной высоты, адаптив
- ✅ Feature-тесты загрузки логов (5 сценариев: импорт, дубли, битый файл, обязательность скриншота, приватность пруфов)

- 🐳 **Docker-окружение вместо Laragon**: `docker compose up -d` поднимает nginx + PHP 8.4 FPM + MySQL 8 + Vite + queue worker. Конфиги в `docker/`, порты настраиваются через `.env`
- 📦 **ADIF-парсер** (`app/Services/Adif/AdifParser.php`): разбор .adi-логов с валидацией CALL/QSO_DATE/TIME_ON/BAND/MODE, извлечением MY_SIG/MY_SIG_INFO и SIG/SIG_INFO (park-to-park), автоконвертацией Windows-1251, лимитами на размер и число записей. Покрыт unit-тестами (`tests/Unit/Services/AdifParserTest.php`)
- 🗄️ **Миграции фундамента v1.2**: таблица `qsos` (связи из логов с дедупликацией и индексом для автозачёта охотников), таблица `activation_proofs` (фото/скриншоты/GPX), поля `user_id`/`adif_path`/`source` в `activations`
- 🧩 Модели `Qso`, `ActivationProof`; константы статусов и новые связи в `Activation`
- 📐 **ARCHITECTURE.md** — полная архитектура UPTA: схема БД, конвейер загрузки логов, экран модерации «за 10 секунд», алгоритм синхронизации с API СРР
- 🤖 CLAUDE.md — контекст проекта для AI-ассистентов

### Изменено

- ⚙️ `vite.config.js`: host/HMR/polling для работы Vite внутри Docker на Windows
- ⚙️ `.env.example`: MySQL по умолчанию (хост `mysql`), порты Docker, локаль `ru`
- 🔧 `composer.json`: скрипты `format`/`format:check` перенесены в секцию `scripts` (раньше лежали в корне и не работали)

### Исправлено

- 🐛 Невалидная структура `composer.json` (ключи вне схемы)
